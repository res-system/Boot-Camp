# Tomcat開発環境構築メモ ## XAMPPインストール ------------------------------------------------------------------------------- [参考](http://blog.outdoorwalk.net/mint-xfce/mint%E3%81%ABxampp%E3%82%92/) 1. ファイルのダウンロード [ダウンロードサイト](https://www.apachefriends.org/jp/download.html) - 対象ファイル xampp-linux-x64-7.0.6-0-installer.run 2. 実行可能にする 「上記ファイル右クリック」＞「プロパティ」＞「アクセス権」＞「プログラムとして実行を許可する」をチェック 3. インストール ``` $ sudo /home/[ユーザ名]/Download/xampp-linux-x64-7.0.6-0-installer.run ``` (「上記ファイル右クリック」＞「rootで実行」でもOK) 4. 「/opt/lampp/」のショートカットを作成。（送る＞サイドペイン※名称は「Lampp」） ## Gradleインストール ------------------------------------------------------------------------------- [参考](http://dukelab.hatenablog.com/entry/2014/06/28/150608) 1. 【eclipseメニュー】＞「ヘルプ」＞「Eclipseマーケットプレース」でウィンドを開き、「検索」で『Gradle』と入力し実行する。 2. 「Gradle IDE Pack」を選択しインストールする。 ※インストールはデフォルトでOKです。 3. 【eclipseメニュー】＞「ファイル」＞「新規」＞「Gradle（STS）」＞「Gradle（STS）Project」でプロジェクト作成を可能です。 4. 【eclipseメニュー】＞「ウィンドウ」＞「ビューの表示」＞「その他」＞「Gradle（STS）」＞「Gradle タスク」を選択する。 5. 「Gradle タスク」ウィンドウで、「プロジェクト」欄で対象のプロジェクトを選択すると実行可能なタスクが表示されます。 ※buildでコンパイルします。 （その他参考） http://qiita.com/pica/items/cbb6160f1ded7bd62abe ## Tomcatインストール ------------------------------------------------------------------------------- 1. ファイルのダウンロード [ダウンロードサイト](http://tomcat.apache.org/) - 対象ファイル apache-tomcat-8.5.16.tar.gz 2. 「/opt/lampp/」の下にファイルを展開(フォルダ名は「tomcat8」) ※管理者権限が必要。 3. 起動スクリプトを作成 ``` $ sudo vi /etc/rc.d/init.d/tomcat ``` ↓ ```/etc/rc.d/init.d/tomcat #!/bin/bash # Tomcat: Start/Stop Tomcat 8.0.35 # # chkconfig: - 90 10 # description: Tomcat is a Java application Server. . /etc/init.d/functions . /etc/sysconfig/network CATALINA_HOME=/opt/lampp/tomcat8 TOMCAT_USER=res LOCKFILE=/var/lock/subsys/tomcat RETVAL=0 start(){ echo "Starting Tomcat: " su - $TOMCAT_USER -c "$CATALINA_HOME/bin/startup.sh" RETVAL=$? echo [ $RETVAL -eq 0 ] && touch $LOCKFILE return $RETVAL } stop(){ echo "Shutting down Tomcat: " $CATALINA_HOME/bin/shutdown.sh RETVAL=$? echo [ $RETVAL -eq 0 ] && rm -f $LOCKFILE return $RETVAL } case "$1" in start) start ;; stop) stop ;; restart) stop sleep 8000 start ;; *) echo $"Usage: $0 {start|stop|restart}" exit 1 ;; esac exit $? ``` ↓ [権限変更] ``` $ sudo chmod 755 /etc/rc.d/init.d/tomcat ``` 4. 「/opt/lampp/tomcat8」の権限変更 ``` $ su $ chown -R res /opt/lampp/tomcat8 $ reboot ``` 5. リモートデバッグ設定 ```/opt/lampp/tomcat8/bin/startup.sh exec "$PRGDIR"/"$EXECUTABLE" start "$@" ↓ exec "$PRGDIR"/"$EXECUTABLE" jpda start "$@" ``` 6. 実行 ``` $ sudo /etc/rc.d/init.d/tomcat start ``` 7. ブラウザで確認 http://localhost:8080/ 8. 停止 ``` $ sudo /etc/rc.d/init.d/tomcat stop ``` ## ランチャーのカスタマイズ ------------------------------------------------------------------------------- デスクトップ下のランチャーに下記のコマンドを追加する。 * Xamppマネージャー - コマンド beesu /opt/lampp/manager-linux-x64.run * Tomcat起動 - コマンド sudo /etc/rc.d/init.d/tomcat start - 「ターミナルで実行する」をチェック * Tomcat停止 - コマンド sudo /etc/rc.d/init.d/tomcat stop - 「ターミナルで実行する」をチェック ## Webアプリケーションデバッグ環境の構築(動的 Web プロジェクト) ------------------------------------------------------------------------------- 1. eclipseを起動。 2. プロジェクト「HelloWorldGradle」を作成する。 1. 【eclipseメニュー】＞「ファイル」＞「新規」＞「動的 Web プロジェクト」で作成する。 2. 【動的 Web プロジェクト】の「プロジェクト名」に『HelloWorldGradle』を入力。 3. 【ビルド・パス上のソース・フォルダー】を『src』(削除)→『src/main/java』に変更し、次へ。 4. 「web.xmlデプロイメント記述子の生成」をチェックし、 「コンテンツ・ディレクトリー」を『WebContent』→『src/main/webapp』に変更して、完了。 3. Gradleプロジェクトへ変換 1. build.gradleファイルを作成する. ```build.gradle apply plugin: 'java' apply plugin: 'war' //追加 repositories { jcenter() } dependencies { providedCompile 'javax.servlet:javax.servlet-api:3.1.0' //追加 } ``` 2. 【プロジェクト右クリックメニュー】＞「構成」＞「Convert to Gradle(STS) Project」を選択します。 3. Gradleのリフレッシュ * 【プロジェクト右クリックメニュー】＞「Gradle(STS)」＞「依存関係のリフレッシュ」を選択します。 * 【プロジェクト右クリックメニュー】＞「Gradle(STS)」＞「すべてリフレッシュ」を選択します。 4. JSP(/src/main/webapp/index.jsp)の作成。 ```/src/main/webapp/index.jsp <%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%> <% String name = "Hello World！ Gradle "; %> <%=name %>

<%=name %>

TEST です。 [<%= new java.util.Date() %>]

``` 5. JSPを実行する。 1. 【Gradleタスクウィンドウ】を開く。 2. プロジェクトを「HelloWorldGradle」に設定。 3. 「build」をクリック。 4. /build/libs/HelloWorldGradle.warが作成されているので、 /opt/lampp/tomcat8/webapps/へwarをコピーする。 5. Tomcat起動。 6. サーバの起動後、ブラウザで確認する - http://localhost:8080/HelloWorldGradle/ 6. Servlet(sample.HelloServlet)の作成。 1. 【プロジェクト右クリックメニュー】＞「新規」＞「サーブレット」を選択します。 2. 下記を入力。 Javaパッケージ：sample クラス名：HelloServlet 7. Servletを実行する。 1. 「5. JSPを実行する。」と同じ手順でTomcat起動までを行う。 2. サーバの起動後、ブラウザで確認する - http://localhost:8080/HelloWorldGradle/HelloServlet 6. eclipseリモートデバッグ設定 1. Tomcat起動。 2. 【eclipseメニュー】＞「実行」＞「デバッグの構成」＞「リモートJavaアプリケーション」＞「新規構成」。 ※ 既に構成を作成されている場合、新規ではなく作成した構成を選択する。 3. 下記の設定を行なって「デバッグ」ボタンを押下する。 プロジェクト：対象のプロジェクト 対象のプロパティ／ホスト：localhost 対象のプロパティ／ポート：8000 4. HelloServletのdoGet()にブレイクポイントを置いて実行する。 ## ライブラリ用プロジェクトの構築(Java プロジェクト) ------------------------------------------------------------------------------- 1. プロジェクト「com-res_system-commons-dao」を作成する。 1. 【eclipseメニュー】＞「ファイル」＞「新規」＞「Java プロジェクト」で作成する。 2. 「プロジェクト名」に『com-res_system-commons-dao』を入力。 3. 【新規ソースフォルダの作成】で『src/main/java』を作成する。 　4. 【デフォルトの出力フォルダ】「com-res_system-commons-dao/build/classes」に変更。 5. 完了。 2. Gradleプロジェクトへ変換 1. build.gradleファイルを作成する. ```build.gradle apply plugin: 'java' repositories { jcenter() } dependencies { // JDBC(mariadb) compile("org.mariadb.jdbc:mariadb-java-client:2.+") // junit testCompile 'junit:junit:4.+' } ``` 2. 【プロジェクト右クリックメニュー】＞「構成」＞「Convert to Gradle(STS) Project」を選択します。 3. Gradleのリフレッシュ * 【プロジェクト右クリックメニュー】＞「Gradle(STS)」＞「すべてリフレッシュ」を選択します。 3. Jarを作成する。 1. 【Gradleタスクウィンドウ】を開く。 2. プロジェクトを「com-res_system-commons-dao」に設定。 3. 「build」をクリック。 4. /build/libs/com-res_system-commons-dao.jarが作成されているので、 必要なところにコピーする。 4. Test用のフォルダを作成する。 1. 『src/test/java』を作成する。 2. 【プロジェクト右クリックメニュー】＞「ビルド・パス」＞「ビルドパスの構成」を選択します。 3. 「ソース」タブを選択し「フォルダーの追加」ボタンを押下します。 4. 「test」フォルダの「java」フォルダを選択します。（「java」フォルダのみです） 5. 「ビルドパスの構成」ダイアログで「ＯＫ」ボタンを押下します。
